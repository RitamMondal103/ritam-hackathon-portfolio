# 🚀 Deployment Guide - Make Your Hackathon Portfolio Public

## 🌟 **Quick Deployment Options**

### **Option 1: GitHub Pages (Free & Easy)**
Best for static websites like your `hackathon-portfolio.html`

#### **Steps:**
1. **Create GitHub Repository**
   ```bash
   git init
   git add .
   git commit -m "Initial commit - Hackathon Portfolio by Ritam Mondal"
   git branch -M main
   git remote add origin https://github.com/yourusername/hackathon-portfolio.git
   git push -u origin main
   ```

2. **Enable GitHub Pages**
   - Go to your repository on GitHub
   - Click **Settings** → **Pages**
   - Select **Deploy from a branch**
   - Choose **main** branch and **/ (root)**
   - Click **Save**

3. **Your Website Will Be Live At:**
   ```
   https://yourusername.github.io/hackathon-portfolio
   ```

### **Option 2: Netlify (Free & Custom Domain)**
Best for custom domains and advanced features

#### **Steps:**
1. **Push to GitHub** (same as Option 1)
2. **Go to Netlify.com**
3. **Sign up with GitHub**
4. **Click "New site from Git"**
5. **Choose your repository**
6. **Build settings:**
   - Build command: `echo "No build needed"`
   - Publish directory: `.` (root)
7. **Deploy site**

### **Option 3: Vercel (Free & Fast)**
Best for performance and modern deployment

#### **Steps:**
1. **Push to GitHub** (same as Option 1)
2. **Go to Vercel.com**
3. **Sign up with GitHub**
4. **Import your repository**
5. **Deploy**

### **Option 4: Custom Domain Setup**
After deploying with any option above:

#### **For GitHub Pages:**
1. **Buy a domain** (GoDaddy, Namecheap, etc.)
2. **In your repo Settings → Pages → Custom domain**
3. **Add your domain** (e.g., `ritammondal.dev`)
4. **Update DNS settings** as instructed

#### **For Netlify/Vercel:**
1. **Go to Domain settings**
2. **Add custom domain**
3. **Update DNS records**
